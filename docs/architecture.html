Copilot Studio & Azure AI Foundry agents behind one governed perimeter — APIM + VNet + private endpoints.

The copilot must read from the core customer-risk system — but the security team requires it can only reach that one approved API, never the open internet, a public data broker, or the database directly.

Allowed vs. blocked (validated)

200Agent calls approved API via APIM (with key)
401Missing / invalid subscription key
400Invalid id — validated at the gateway
404Undefined route / external API — no connector
403Direct-to-backend bypass — private only
429Rate limit exceeded (>20 / 60s)

Why it holds

VNetEverything past APIM is private — no public access, deny-internet egress
APIMOne governed door; enforces auth, rate-limit, validation, logging
DLPCopilot Studio (SaaS) locked to the approved connector only
RBACManaged identity + scoped roles — no shared secrets in Foundry
DataFoundry agent state stays in customer-owned Cosmos / Search / Storage
